❯ II. METHODS                                                                   
                                                                                
Fig. 2 presents the overall framework of the proposed method[cite: 212]. As     
shown in Fig. 2 (a), surgical videos are visually decomposed into foreground    
instrument manipulations, middle ground tissue structures, and background       
anatomical context[cite: 213]. Expert surgeons typically demonstrate: (1)       
coherent and efficient instrument trajectories[cite: 214]; (2) adjustment of    
endoscope angles and procedural steps to minimize interference from instrument  
occlusion, bleeding, or smoke, thereby maintaining a clear surgical field       
[cite: 215]; and (3) precise force application to avoid excessive tissue        
traction[cite: 216]. Based on this, instrument manipulation, tissue handling,   
and surgical field clarity are defined as the three core criteria for SSA[cite: 
 217]. As illustrated in Fig. 2 (b), existing methods face several key          
challenges: single-video approaches often fail to capture subtle inter-video    
differences, and prior studies lack fine-grained feature analysis[cite: 218].   
To address these limitations, a contrastive learning-based approach is          
proposed, extracting instrument, tissue, and spatiotemporal features to enable  
fine-grained SSA[cite: 219]. Moreover, to reduce the "black-box" nature of      
existing SSA methods, Class Activation Mapping (CAM) visualizations are applied 
 to the extracted features during scoring, further enhancing model              
interpretability[cite: 220]. The fine-grained contrastive learning framework in 
 Fig. 2 (c) uses the highest-scoring video as reference and the video under     
evaluation as query[cite: 221]. The I3D network is utilized as the backbone to  
extract the overall spatiotemporal features[cite: 222]. A ResNet-34-based       
static feature extractor captures key tissue regions and the static global      
features of the surgical field[cite: 223]. Simultaneously, an instrument        
segmentation network is incorporated, integrating segmentation masks with       
spatiotemporal features to focus on dynamic instrument manipulation[cite: 224]. 
 Once the fine-grained features are obtained, differences between the reference 
 and query videos are computed across two dimensions: static global features    
and dynamic instrument features[cite: 225]. Finally, a contrastive regressor is 
 employed to map both the difference features and the score differences between 
 the video pair, enabling high-precision skill regression with strong ranking   
consistency[cite: 226].                                                         
                                                                                
A. Problem Formulation                                                          
Given a surgical video $V\in\mathbb{R}^{T\times H\times W\times C}$ annotated   
with a skill score $y$, where $T$, $H$, $W$ and $C$ denote the clip length,     
height, width, and number of channels respectively, surgical video skill        
assessment has been commonly formulated as a regression problem by taking the   
video as input and predicting the corresponding skill score as output[cite:     
228]:                                                                           
$y=\mathcal{R}(\mathcal{E}(V))$ [cite: 229]                                     
Where $\mathcal{E}(\cdot)$ and $\mathcal{R}(\cdot)$ represent the feature       
extractor and the regressor network, respectively[cite: 231]. However, holistic 
 video-based methods exhibit limitations in capturing subtle distinctions       
between surgical videos, particularly when the videos possess similar skill     
scores[cite: 232]. To overcome this limitation, we propose a fine-grained       
contrastive regression framework that reformulates SSA as a regression problem  
based on the skill score difference between a query video                       
$V_{q}\in\mathbb{R}^{T\times H\times W\times C}$ [cite: 233] and a reference    
video $V_{r}\in\mathbb{R}^{T\times H\times W\times C}$[cite: 277]. The          
framework aims to enhance the sensitivity and accuracy of skill score           
prediction by leveraging contrastive cues extracted from paired videos[cite:    
277]. Specifically, each input consists of a video pair, with one serving as    
the query video $V_{q}$ and the other as a reference video $V_{r}$ associated   
with a known skill score $y_{r}$[cite: 278]. By explicitly contrasting subtle   
variations across key dimensions-such as manipulation of foreground surgical    
instruments, middle ground tissue state variations, and surgical field          
clarity-the model can more effectively learn discriminative local features that 
 strongly correlate with skill scores, thereby enabling more precise regression 
 of the query video's score $y_{q}$[cite: 279]. This regression task is         
formally expressed as:                                                          
$y_{q}=\mathcal{CR}(\mathcal{E}(V_{q}),\mathcal{E}(V_{r}))+y_{r}$ [cite: 281]   
Here, $\mathcal{CR}$ denotes the contrastive regressor[cite: 283]. During the   
training phase, $V_{q}$ and $V_{r}$ are used as inputs, the skill score         
difference between the two surgical videos is regressed, and the network        
parameters are optimized by minimizing the discrepancy between the predicted    
and ground-truth score differences[cite: 283].                                  
                                                                                
B. High-Quality Surgical Instrument Mask Dataset Construction via SAM2          
SAM2 is a large-scale model designed for general image and video segmentation   
tasks and demonstrates outstanding performance in zero-shot segmentation        
scenarios[cite: 285]. Even without task-specific training, SAM2 is capable of   
recognizing and segmenting arbitrary targets in both images and videos[cite:    
286]. Leveraging SAM2, we constructed a high-quality surgical instrument        
segmentation dataset, encompassing both simulated environments and clinical     
scenarios (including cholecystectomy and hernia repair surgeries), comprising a 
 total of 169,139 annotated frames, making it the largest surgical instrument   
mask dataset specifically designed for SSA tasks to date[cite: 287]. As shown   
in Fig. 3 (b), the overall framework of SAM2 primarily consists of five core    
components: the Image Encoder, Prompt Encoder, Mask Decoder, Mask Attention     
module, and Memory Bank[cite: 288]. As illustrated in Fig. 3, the dataset       
construction pipeline is as follows: first, the initial frame of each surgical  
video is interactively annotated, in which positive points within instrument    
regions and negative points in non-instrument regions are selected as initial   
prompts[cite: 289]. Subsequently, the surgical video frames, together with the  
corresponding prompts, are input into the SAM2 model[cite: 290]. The prompt     
embeddings and image feature embeddings are then respectively provided to the   
Memory Attention module and the Mask Decoder[cite: 291]. Specifically, the      
Memory Attention module exploits historical frame information stored in the     
memory bank to enhance the feature representation of the current frame through  
self-attention and cross-attention mechanisms, while the Mask Decoder           
integrates the prompt embeddings with the Memory Attention output to generate   
candidate masks together with their corresponding IoU confidence scores[cite:   
292]. For each target, the mask with the highest IoU score is retained as the   
final segmentation result[cite: 293]. The generated masks and their associated  
feature embeddings are subsequently written into the memory bank to support the 
 segmentation of subsequent frames[cite: 294, 296]. The memory bank stores      
recent frame features using a FIFO strategy to improve temporal                 
consistency[cite: 296]. Finally, a manual quality inspection was performed on   
the masks automatically generated by SAM2[cite: 297]. For frames exhibiting     
inferior segmentation quality, as illustrated in Fig. 3 (c), meticulous manual  
refinement was conducted, resulting in high-quality mask annotations, as shown  
in Fig. 3 (d)[cite: 298]. These annotations provide reliable data support,      
enabling the model to accurately focus on surgical instrument operations[cite:  
299].                                                                           
                                                                                
C. Dynamic Instrument Temporal Feature Extraction Module                        
1) Short-term Spatiotemporal Feature Extraction: Given a paired query video     
$V_{q}$ and reference video $V_{r}$, each video is divided into N overlapping   
snippets using a sliding window with a stride of 10, where each snippet         
containing 16 consecutive frames $V=\{X_{i}\}_{i=1}^{N}$[cite: 301]. Here,      
$X_{i}$ with the size of $C\times T\times H\times W$[cite: 301]. This           
overlapping strategy preserves temporal continuity while improving the          
robustness of feature extraction[cite: 301]. To effectively capture complex     
spatiotemporal dynamics, I3D is employed as the backbone network for dynamic    
visual feature extraction[cite: 302]. I3D extends 2D convolution kernels into   
3D, thereby processing spatial and temporal information simultaneously[cite:    
303]. Mixed convolution layers are incorporated to further expand the temporal  
receptive field[cite: 304]. Following the mixed convolution layers, a           
max-pooling operation is applied along the spatial axis, which is formulated as 
 follows[cite: 305]:                                                            
$F_{clip}(X_{i}) = maxpool (Conv_{mix}(X_{i}))$ [cite: 306]                     
Here, $Conv_{mix}$ denotes the mixed convolution layers, and $maxpool$          
represents the max-pooling layer[cite: 308]. The output                         
$F_{clip}(X_{i})\in\mathbb{R}^{C_{1}\times T_{1}\times W_{1}\times H_{1}}$      
represents the short-term local spatiotemporal features extracted from each     
video snippet[cite: 309].                                                       
                                                                                
2) Mask-guided Attention Module: To enable more precise focus on instrument     
manipulation in the foreground, an independent mask-guided attention module was 
 designed in this study[cite: 310]. The module was constructed on top of the    
segmentation network and operates in parallel with the I3D network for          
spatiotemporal feature extraction, specifically designed to capture             
instrument-related features with high discriminative capability[cite: 311].     
Unlike conventional segmentation tasks that primarily aim to generate explicit  
binary masks, the attention module emphasizes the identification of regions of  
greater importance at the feature level[cite: 312]. Specifically, the module    
transforms high-dimensional instrument semantic features into continuous-valued 
 spatial attention maps, thereby explicitly quantifying the relative importance 
 of individual pixel regions and effectively guiding subsequent feature         
learning[cite: 313]. Without modifying the original segmentation structure,     
this mechanism further enhances the representation of critical operational      
regions, thereby improving the discriminative performance in SSA tasks[cite:    
314]. In practice, the module receives video snippets $X_{i}$ as input and,     
based on instrument mask features generated by the segmentation network,        
constructs the corresponding mean and maximum spatial attention maps via        
average and max pooling operations, respectively, as follows[cite: 315, 316]:   
$\mathcal{A}_{mean}^{t,w,h}=\frac{1}{C_{2}}\sum_{c=1}^{C_{2}}(F_{ms}(X_{i}))_{c 
}^{t,w,h}$ [cite: 317]                                                          
$\mathcal{A}_{max}^{t,w,h}=\max_{c}(F_{ms}(X_{i}))_{c}^{t,w,h}$ [cite: 318]     
Here, $t$ denotes the temporal index, $c$ the $c$-th channel, and $w$ and $h$   
the spatial coordinates of the feature map[cite: 319]. The input features       
$F_{ms}$ are the instrument features extracted by the segmentation              
network[cite: 320]. To enable the network to adaptively learn and update the    
weights between the mean $\mathcal{A}_{mean}$ and maximum $\mathcal{A}_{max}$   
attention maps, an aggregation function $(f_{Agg})$ is applied to combine them  
into a single-channel attention map $\mathcal{A}$[cite: 321]. Subsequently, an  
activation function $(\sigma)$ is used to constrain the values of the           
aggregated attention map within the range $[0,1]$[cite: 321]. This process can  
be formulated as follows[cite: 322]:                                            
$\mathcal{A}=\sigma(f_{Agg}(\mathcal{A}_{mean},\mathcal{A}_{max}))$ [cite: 323] 
                                                                                
3) Attention Refinement Via Instrument Masks: To guide the model to focus on    
the instrument manipulation region, a high-quality instrument mask dataset was  
generated using the SAM2 model[cite: 325]. These masks constituted a            
high-quality ground-truth dataset[cite: 326]. Instead of being directly used    
frame by frame, a lightweight temporal smoothing operation was applied to       
mitigate jitter and local instability inherent in single-frame masks for each   
video frame, providing effective supervision for the attention map              
$\mathcal{A}$[cite: 326]. Specifically, to mitigate jitter and local            
instability inherent in single-frame masks, the masks of two adjacent frames    
were summed[cite: 326]. The resulting mask was then resized to match the        
attention map dimensions using a 2D average pooling operation, expressed        
as[cite: 327]:                                                                  
$M_{gt}^{w,h}=f_{2D}\left(\sum_{t=1}^{T/2}(M^{2t-1,w,h}+M^{2t,w,h})\right)$     
[cite: 328]                                                                     
After aligning the sizes of the ground-truth mask $M_{gt}^{w,h}$ and the        
attention map $\mathcal{A}$ $(T_{2}\times W_{2}\times H_{2})$ a pixel-level     
loss function was introduced to constrain the attention map $\mathcal{A}$,      
thereby progressively enhancing the quality of the attention map[cite: 330]:    
$Loss_{mask}=\frac{1}{T\times W\times H}\sum_{t=1}^{T}\sum_{w=1}^{W}\sum_{h=1}^ 
{H}||\mathcal{A}^{t,w,h}-M_{gt}^{t,w,h}||^{2}$ [cite: 331]                      
Here, $w$ and $h$ denote the spatial coordinates in the mask and attention      
maps, and $t$ represents the temporal index[cite: 333]. This loss function      
allows the attention map produced by the mask-guided attention module to more   
accurately concentrate on the instrument manipulation region[cite: 334].        
Finally, the attention map is multiplied element-wise with the short-term local 
 features to enhance the representation of the instrument manipulation          
region[cite: 335, 336]:                                                         
$F_{dy}(X_{i})=F_{clip}(X_{i})\odot(\mathcal{A}+\mathcal{I})$ [cite: 338]       
Here, $\odot$ indicates the element-wise multiplication operator, and           
$\mathcal{I}$ denotes a matrix with all elements equal to one[cite: 340]. This  
strategy substantially enhances features of the instrument manipulation region, 
 yielding a more discriminative representation for subsequent SSA[cite: 341].   
                                                                                
D. Intra-Video Context Modeling Module                                          
In addition to dynamic instrument operations, changes in tissue state and the   
clarity of the surgical field are also regarded as critical dimensions in the   
SSA task[cite: 344]. Specifically, insufficiently skilled surgeons may cause    
tissue damage or intraoperative bleeding due to inappropriate traction, thereby 
 increasing the risk of postoperative complications[cite: 345]. In contrast,    
highly skilled surgeons are generally able to adjust the endoscope angle and    
optimize their operative strategies to minimize the impact of fogging,          
bleeding, and smoke generated by tissue cauterization on the visibility of the  
surgical field[cite: 346]. These efforts help maintain clear visualization of   
anatomical structures and consequently enhance surgical safety[cite: 347].      
Therefore, reliance solely on instrument dynamics is insufficient to            
comprehensively reflect a surgeon's actual skill level[cite: 348]. To address   
this limitation, an intra-video contextual modeling module is introduced to     
supplement the representation of appearance-related information, such as tissue 
 state changes and surgical field clarity, thereby providing more comprehensive 
 and reliable discriminative evidence for subsequent SSA[cite: 349]. Compared   
with the I3D network, the ResNet module offers superior capability in           
extracting global appearance information from video frames[cite: 350].          
Therefore, ResNet-34 was adopted as the backbone network for this               
component[cite: 351]. Specifically, this component comprises a ResNet-34 module 
 $(\mathcal{R})$ and a multi-branch downsampling module                         
$(\mathcal{D}_{mul})$[cite: 352]. First, high-level global features of the      
video clips are extracted using the ResNet module, as formulated below[cite:    
353]:                                                                           
$F_{global} = \mathcal{R}(X_{i}), i = 1, 2, ..., N$ [cite: 354]                 
In this formulation, the input $(X_{i})$ represents a video clip[cite: 356]. To 
 further capture local details, a multi-branch downsampling module              
$(\mathcal{D}_{mul})$ is introduced[cite: 357]. The corresponding formulation   
is expressed as follows[cite: 357]:                                             
$F_{mul} = \mathcal{D}_{mul}(F_{global}), s \in \{2,4,8\}$ [cite: 358]          
Here, $s$ denotes the downsampling scale[cite: 360]. A larger downsampling      
scale enables the capture of macroscopic contextual variations, such as overall 
 tissue trends and instrument states[cite: 360]. Conversely, a smaller          
downsampling scale facilitates the capture of subtle tissue variations[cite:    
361]. Through this design, the model preserves global perceptual capability     
while also maintaining sensitivity to local variations, thereby providing       
richer discriminative information for skill assessment and further enhancing    
the model's evaluative capacity[cite: 362]. Ultimately, the features obtained   
from the three downsampling branches at different scales are concatenated to    
form a multi-scale representation, as expressed by the following equation[cite: 
 363]:                                                                          
$F_{sta} = Concat(\mathcal{D}_{mul}^{2}(F_{global}),                            
\mathcal{D}_{mul}^{4}(F_{global}), \mathcal{D}_{mul}^{8}(F_{global}))$ [cite:   
364]                                                                            
Up to this stage, more detailed and stable global appearance representations of 
 video frames are obtained by the intra-video contextual modeling module,       
thereby further enhancing its capability to perceive tissue state variations    
and the overall clarity of the surgical field[cite: 365].                       
                                                                                
E. Fine-Grained Contrastive Learning Framework                                  
In SSA tasks, videos with similar scores often exhibit highly similar operating 
 procedures, resulting in only subtle variations in skill performance[cite:     
367]. Most existing approaches predict skill scores on individual videos        
independently, which restricts their ability to capture fine-grained            
differences across videos[cite: 368]. This limitation is particularly           
pronounced in SSA tasks where exceptional precision is required[cite: 369].     
Nevertheless, these fine-grained distinctions are crucial for achieving         
accurate and reliable SSA[cite: 370]. To address this challenge, a fine-grained 
 contrastive learning framework is proposed to enhance the model's capacity to  
perceive and model subtle differences between surgical videos[cite: 371]. The   
component primarily consists of a contrastive score decoder $(\mathcal{D})$ and 
 a contrastive score regressor $(\mathcal{CR})$[cite: 372]. By leveraging the   
transformer's sequence-to-sequence modeling capability in conjunction with      
cross-attention mechanisms, semantic associations are established between       
dynamic and static features in the reference and query videos, thereby          
effectively capturing their fine-grained differences[cite: 373]. The learned    
differential features are subsequently input into the score regressor, which    
produces a relative score indicating the skill performance disparity between    
the two videos[cite: 374]. Finally, the final skill rating of the query video   
is obtained by adding this differential score to the skill rating of the        
reference video[cite: 375].                                                     
                                                                                
1) Contrastive Score Decoder: To effectively capture fine-grained differences   
between the query video $V_{q}$ and reference video $V_{r}$, thereby enabling a 
 more accurate assessment of surgical skill, a Contrastive Score Decoder was    
designed[cite: 376]. This decoder employs a multi-head cross-attention          
mechanism to simultaneously model similarities and dissimilarities between the  
two sequences, producing more discriminative feature representations for        
subsequent score estimation[cite: 377]. Specifically, leveraging the dynamic    
instrument temporal feature extraction module and the intra-video contextual    
modeling module, the static global features $(F_{sta})$ and the dynamic         
instrument operation features $(F_{dy})$ are extracted for both the reference   
and query videos[cite: 378]. To preserve the independent information contained  
in these two feature types, they are each fed into separate decoders            
$(\mathcal{D})$[cite: 379]. In this setup, the reference video features         
$(F_{sta}^{r}$ and $F_{dy}^{r})$ serve as the memory (keys and values) for      
their respective decoders, whereas the query video features $(F_{sta}^{q}$ and  
$F_{dy}^{q}$) function as the corresponding queries[cite: 380]. By employing    
cross-attention learning, the differential representations $(F_{sta}^{diff}$    
and $F_{dy}^{diff})$ of the static global features and dynamic instrument       
operation features are computed independently[cite: 381]:                       
$F_{sta}^{diff} = MultiHeadAtten (Q_{sta}, K_{sta}, V_{sta})$ [cite: 382]       
$F_{dy}^{diff} = MultiHeadAtten (Q_{dy}, K_{dy}, V_{dy})$ [cite: 382]           
In this process, the static global features $(F_{sta})$ and the dynamic         
instrument operation features $(F_{dy})$, denoted as $Q$, $K$, and $V$, are     
generated by projecting the input features into intermediate tensors using      
three learnable linear transformation matrices, $\mathcal{W}_{Q}$,              
$\mathcal{W}_{K}$, and $\mathcal{W}_{V}$, respectively[cite: 385]. Here, for    
simplicity, only the projections of the dynamic instrument operation features   
$F_{dy}$ are shown[cite: 385]. It should be emphasized that all subsequent      
processing steps are applied to the static global features $F_{sta}$ in exactly 
 the same manner[cite: 388].                                                    
$Q_{dy}=\mathcal{W}_{Q}F_{dy}^{q}$ [cite: 389]                                  
$K_{dy}=\mathcal{W}_{K}F_{dy}^{r}$ [cite: 389]                                  
$V_{dy}=\mathcal{W}_{V}F_{dy}^{r}$ [cite: 390]                                  
Here, $F_{dy}^{q}$ denotes the dynamic instrument operation features of the     
query video, while $F_{dy}^{r}$ represents the corresponding features of the    
reference video[cite: 391]. The attention weights are then obtained through a   
softmax operation and subsequently applied to the value matrix[cite: 392]:      
$Atten(Q_{dy},K_{dy},V_{dy})=softmax\left(\frac{Q_{dy}K_{dy}^{T}}{\sqrt{d_{k}}} 
\right)V_{dy}$ [cite: 393]                                                      
Here, $d_{k}$ denotes the hidden dimensionality for queries and keys and is     
introduced to prevent gradient vanishing[cite: 394]. This process can be        
executed in parallel across multiple attention heads, resulting in the weight   
matrix of each attention head, denoted as head[cite: 395]:                      
$head_{i}=Atten(Q_{dy}^{i},K_{dy}^{i},V_{dy}^{i})$ [cite: 403]                  
$MultiHead(Q_{dy},K_{dy},V_{dy})=Concat(head^{1},...,head^{m})\mathcal{W}^{O}$  
[cite: 403]                                                                     
In this study, $m=8$[cite: 404]. Here, $\mathcal{W}^{O}$ denotes the learnable  
weight matrix of the final output layer[cite: 404]. The resulting multi-head    
representations, $MultiHead(Q_{dy},K_{dy},V_{dy})$ and                          
$MultiHead(Q_{sta},K_{sta},V_{sta})$, correspond to the differential encodings  
of the dynamic instrument operation features and static global features (i.e.,  
$F_{dy}^{diff}$ and $F_{sta}^{diff}$), effectively capturing the fine-grained   
differences between the query video and the reference video[cite: 405].         
                                                                                
2) Contrastive Score Regressor: Based on the differential features generated by 
 the contrastive score decoder, a MLP is introduced to learn the relative       
scores between video pairs, thereby quantifying the skill disparity between the 
 query video and the reference video[cite: 406]. First, the differential        
representations of the query and reference videos in terms of the static global 
 features and dynamic instrument operation features are obtained (denoted as    
$F_{sta}^{diff}$ and $F_{dy}^{diff}$ respectively)[cite: 407]. These two types  
of differential features are then fused to obtain a unified differential        
feature representation $F^{diff}$[cite: 408]. Finally, the fused differential   
feature $F^{diff}$ is fed into the contrastive score regressor $\mathcal{CR}$,  
whose output corresponds to the score difference $\triangle s$ between the      
video pairs[cite: 408]. When $\Delta s>0$ it indicates that the skill score of  
the query video is higher than that of the reference video; otherwise, the      
skill score is lower than or equal to that of the reference video[cite: 409,    
410]. Accordingly, the final score of the query video can be expressed as[cite: 
 411]:                                                                          
$y_{q}=y_{r}+\mathcal{CR}(\mathcal{D}(V_{q},V_{r}))=y_{r}+\triangle s$ [cite:   
412]                                                                            
Here, $y_{r}$ represents the skill score of the reference video[cite: 413]. The 
 following loss function is employed to minimize the discrepancy between the    
ground-truth score difference $\triangle s$ of video pairs and the difference   
predicted $\hat{\triangle}s$ by the MLP regressor[cite: 414]:                   
$Loss_{score}=||\hat{\triangle}s-\triangle s||^{2}$ [cite: 415]III.             
EXPERIMENTAL SETUP                                                              
                                                                                
A. Datasets                                                                     
1) Simulated Dataset: The simulated dataset employed in this study is the       
JIGSAWS dataset [9], a widely used benchmark for SSA tasks, collected with the  
da Vinci surgical system and comprising three core robotic surgical tasks: KT,  
SU, and NP. As illustrated in Fig. 1 (b), eight surgeons with varying           
experience levels were recruited, including two experts, two intermediates, and 
 four novices. Each surgeon performed each task five times, yielding 36 KT      
videos, 28 SU videos, and 39 NP videos, together with corresponding             
synchronized kinematic data. As shown in Fig. 1 (c), all videos were            
independently evaluated by multiple expert raters using the OSATS criteria,     
encompassing six key skill domains: Respect for Tissue, Suture/Needle Handling, 
 Time and Motion, Instrument Handling, Flow of Operation, and Overall           
Performance. The average of these six scores was adopted as a global indicator  
of surgical proficiency, referred to as the Global Rating Score (GRS), with     
values ranging from 6 to 30. As depicted in Fig. 1 (a), the mean durations of   
the KT, SU, and NP videos were 57.3 s, 112.6 s, and 108.6 s, respectively. To   
comprehensively evaluate the proposed model, three standard experimental        
settings provided in the dataset were adopted: LOUO, LOSO, and random 4-Fold    
cross-validation. In the LOUO setting, 8-Fold cross-validation was conducted,   
with all samples from a single user sequentially assigned to the test set while 
 the remaining users data were used for training. In the LOSO setting, each     
trial of a given user was sequentially used as the test set, with the remaining 
 trials forming the training set, resulting in a 5-Fold cross-validation        
scheme. Additionally, a random 4-Fold cross-validation was performed to assess  
the overall generalization capability of the model. The study strictly adhered  
to the experimental protocols defined by the JIGSAWS dataset, ensuring fair     
comparisons with existing methods.                                              
                                                                                
2) Clinical Cholecystectomy Dataset: Clinical laparoscopic cholecystectomy data 
 were sourced from the HeiChole dataset [36], collected across three surgical   
centers in Germany, comprising 24 complete procedure recordings, each           
accompanied by surgical skill annotations. Each video captures the entire       
procedure, spanning from the initial insertion of the laparoscope into the      
patient's abdomen to its final withdrawal. To ensure standardized and           
consistent evaluation, all videos were scored independently by three trained    
medical students. The scoring was guided by a modified version of the GOALS     
[35], covering five dimensions: depth perception (S1), bimanual dexterity (S2), 
 efficiency (S3), tissue handling (S4), and operational difficulty (S5). Each   
dimension was rated on a scale of 1 to 5, yielding total scores ranging from 5  
to 25. As depicted in Fig. 1 (a), the two key surgical phases in the            
cholecystectomy videos averaged 1031.2 s and 400.7 s, respectively, with skill  
scores spanning 15 to 21. Prior research has commonly concentrated on two       
critical sub-stages of cholecystectomy for skill assessment: Calot triangle     
dissection and Gallbladder dissection. Following this established framework     
[34] [21] [1], the proposed method utilizes randomized 4-Fold cross-validation  
to evaluate model performance.                                                  
                                                                                
3) Clinical Hernia Repair Suturing Surgery Dataset: The clinical hernia repair  
suturing dataset utilized in this study was sourced from our private            
collection. Given the similarity of this procedure to laparoscopic              
cholecystectomy, as both are performed within the abdominal cavity, the GOALS   
metric was adopted for evaluation. To ensure annotation accuracy and            
consistency, the videos were evaluated independently by two experienced         
surgeons using this metric. Videos with concordant scores were retained,        
whereas discrepancies were resolved through discussion to achieve consensus. As 
 illustrated in Fig. 1 (a), the dataset comprises 34 surgical videos, with an   
average duration of 188.1 s and skill scores ranging from 15 to 21. Randomized  
4-Fold cross-validation was employed to evaluate model performance on this      
dataset. In summary, the generalization capability of the proposed model was    
systematically evaluated using both simulated and clinical datasets. The        
simulated evaluation utilized the JIGSAWS dataset, which includes three         
fundamental robotic surgical tasks: KT, SU, and NP. The clinical evaluation     
comprised two types of laparoscopic procedures: cholecystectomy from the        
HeiChole dataset and hernia repair suturing from a private dataset. The         
clinical datasets contained 48 and 34 surgical videos, respectively, each       
scored using the GOALS metric, specifically designed for laparoscopic surgery,  
with total scores ranging from 15 to 21. Collectively, these datasets encompass 
 varying surgical types, video lengths, and procedural complexities, allowing a 
 thorough evaluation of the model's performance across diverse scenarios and    
providing a robust foundation for assessing its reliability and generalization. 
                                                                                
B. Evaluation Metrics                                                           
The MAE and SRCC metrics are employed as evaluation metrics to assess the       
proposed model's performance on the SSA task. SRCC quantifies the monotonic     
relationship between predicted and actual scores, while MAE measures the        
average error between predicted and true scores.                                
                                                                                
1) Spearman Rank Correlation Coefficient (SRCC): In accordance with prior skill 
 assessment methods, the SRCC is utilized to evaluate the performance of the    
proposed model. This metric assesses the Pearson correlation coefficient        
between two rank vectors, $p$ and $q$, which represent the predicted scores and 
 the ground-truth scores, respectively. The SRCC is defined as follows:         
$SRCC=\frac{\sum_{i=1}^{n}(p_{i}-\overline{p})(q_{i}-\overline{q})}{\sqrt{\sum_ 
{i=1}^{n}(p_{i}-\overline{p})^{2}\sum_{i=1}^{n}(q_{i}-\overline{q})^{2}}}$      
Here, $p$ and $q$ represent the ranking of the $i$-th sample in the predicted   
and ground-truth sequences, respectively, and $\overline{p}, \overline{q}$ are  
their corresponding mean rankings. The SRCC ranges from -1 to 1                 
($SRCC\in[-1,1]$), with higher values indicating a stronger positive            
correlation and consequently better model performance.                          
                                                                                
2) Mean Absolute Error (MAE): MAE is a statistical metric used to quantify the  
discrepancy between predicted and ground-truth scores. The average magnitude of 
 prediction errors is computed by taking the mean of the absolute differences   
between predicted and true scores across the test set. The formula is defined   
as follows:                                                                     
$MAE=\frac{1}{n}\sum_{i=1}^{n}|\hat{y}_{i}-y_{i}|$                              
Here, $MAE \in [0, +\infty)$. In contrast to most existing methods that report  
only the SRCC metric, the MAE metric has been additionally incorporated to      
provide a more intuitive reflection of skill score regression accuracy.

  为什么需要插值？

  根本原因：SAM3掩膜和I3D特征的尺寸不匹配

  尺寸不匹配的原因：

  1. SAM3掩膜 (从磁盘加载):
    - 尺寸: (B, T_mask, H_mask, W_mask)
    - 例如: (B, 16, 224, 224) — 与原始视频尺寸相同
  2. I3D动态特征 (网络输出):
    - 尺寸: (B, C, T, H, W)
    - 例如: (B, 832, 8, 7, 7) — 经过多次卷积和池化后变小了
  3. 点乘操作的要求:
    - 掩膜需要与I3D特征点乘: features * masks
    - 两个张量在空间维度上必须尺寸完全一致

  I3D特征尺寸变小的过程：

  输入视频: (B, 3, 16, 224, 224)
      ↓
  Stem conv: 7x7 stride=2 → (B, 64, 16, 112, 112)
      ↓
  MaxPool: stride=2 → (B, 64, 16, 56, 56)
      ↓
  MaxPool: stride=2 → (B, 192, 16, 28, 28)
      ↓
  MaxPool3D: stride=2 → (B, 256, 8, 14, 14)
      ↓
  MaxPool: stride=2 → (B, 480, 8, 7, 7)  ← 最终特征尺寸

  所以 (224, 224) 的掩膜必须被缩小到 (7, 7) 才能与特征点乘。

  trilinear vs nearest 的区别

  ┌───────────┬────────────┬───────────────┬──────────────────────────┐
  │ 插值模式  │ 输出值范围 │   适合场景    │     对二值掩膜的影响     │

  │ trilinear │ 连续 [0,1] │ 浮点特征图    │ 产生中间值（0.3, 0.7等） │
  ├───────────┼────────────┼───────────────┼──────────────────────────┤
  │ nearest   │ 离散 {0,1} │ 二值/类别数据 │ 保持二值性               │
  └───────────┴────────────┴───────────────┴──────────────────────────┘

  # 当前代码
  mode='trilinear'  # 对二值掩膜(0/1)会产生中间值

  - trilinear插值: 边界处产生 0.2, 0.5, 0.8 等值
  - nearest插值: 保持严格为 0 或 1

  def _load_masks(self, video_id):
      if not self.use_mask:
          return None

      mask_path = os.path.join(self.mask_dir, video_id)

      if self.mask_format == 'npy':
          try:
              masks = np.load(mask_path + '_masks.npy')
              return torch.from_numpy(masks).float()
          except FileNotFoundError:
              print(f"Warning: Mask file not found: {mask_path}_masks.npy")
              return None
      else:
          mask_dir = os.path.join(self.mask_dir, video_id)
          if os.path.isdir(mask_dir):
              # Try multiple filename patterns for better compatibility
              # Pattern 1: *_mask.png (SAM3 output)
              mask_files = sorted(glob.glob(os.path.join(mask_dir,
  '*_mask.png')))
              if len(mask_files) == 0:
                  # Pattern 2: *.png (any PNG file)
                  mask_files = sorted(glob.glob(os.path.join(mask_dir,
  '*.png')))
              if len(mask_files) == 0:
                  # Pattern 3: *.jpg (JPEG files)
                  mask_files = sorted(glob.glob(os.path.join(mask_dir,
  '*.jpg')))

              if len(mask_files) == 0:
                  print(f"Warning: No mask files found in {mask_dir}")
                  return None

              masks = []
              for mask_file in mask_files[:self.clip_length]:
                  mask = cv2.imread(mask_file, cv2.IMREAD_GRAYSCALE)
                  if mask is None:
                      print(f"Warning: Failed to read mask file: {mask_file}")
                      continue
                  mask = mask / 255.0
                  masks.append(mask)

              if len(masks) > 0:
                  masks = np.stack(masks, axis=0)
                  return torch.from_numpy(masks).float()
              else:
                  print(f"Warning: No valid masks loaded for video {video_id}")

      return None

  意义:
  - 添加了异常处理：npy文件找不到时优雅处理而非崩溃
  - 支持多种掩膜文件名格式：
    - *_mask.png (SAM3默认输出)
    - *.png (任意PNG文件)
    - *.jpg (JPEG格式)
  - 添加了详细的错误日志，便于调试
  - 检测并报告掩膜读取失败的情况
